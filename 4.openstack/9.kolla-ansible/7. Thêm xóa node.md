# Thêm node
Add host với node mới `/etc/hosts`

    10.2.2.211 compute5

Thêm public key để truy cập node mới từ node deploy:

    ssh-copy-id root@compute5

Thêm compute5 vào group [compute] cho file multinode:

    [compute]
    compute5

Chạy bootstrap-servers cho compute5

    (venv) root@deploy:~# kolla-ansible bootstrap-servers -i multinode
    --limit compute5

Chỉ chạy bootstrap trên server mới, bootstrap trên những server đang chạy
sẽ có những rủi ro nhất định. Nhưng chính vì không chạy lại được bootstrap trên
những server cũ, node mới sẽ không được thêm vào file hosts trên các node cũ.

Cách xử lý là thêm thủ công vào /etc/hosts cho các node

Pull image về compute5

    (venv) root@deploy:~# kolla-ansible pull -i multinode --limit compute5

Deploy compute5

    (venv) root@deploy:~# kolla-ansible deploy -i multinode --limit compute5

# Xóa node controller
Cần di chuyển các tài nguyên trong node trước khi xóa

Tìm neutron routers trên node và di chuyển chúng. Disable L3 agent:

    l3_id=$(openstack network agent list --host <host> --agent-type l3 -f value -c ID)
    target_l3_id=$(openstack network agent list --host <target host> --agent-type l3 -f value -c ID)
    openstack router list --agent $l3_id -f value -c ID | while read router; do
        openstack network agent remove router $l3_id $router --l3
        openstack network agent add router $target_l3_id $router --l3
    done
    openstack network agent set $l3_id --disable

Tiếp tục với dhcp agents:

    dhcp_id=$(openstack network agent list --host <host> --agent-type dhcp -f value -c ID)
    target_dhcp_id=$(openstack network agent list --host <target host> --agent-type dhcp -f value -c ID)
    openstack network list --agent $dhcp_id -f value -c ID | while read network; do
        openstack network agent remove network $dhcp_id $network --dhcp
        openstack network agent add network $target_dhcp_id $network --dhcp
    done

Dừng toàn bộ dịch vụ trên node cần xóa:

    kolla-ansible stop -i <inventory> --yes-i-really-really-mean-it [ --limit <limit> ]

Xóa node khỏi inventory `multinode`

Reconfigure các controller node còn lại để cập nhật các node của cluster ví dụ như MariaDB and RabbitMQ

    kolla-ansible deploy -i <inventory> [ --limit <limit> ]

Xóa các dịch vụ của node xóa:

    openstack network agent list --host <host> -f value -c ID | while read id; do
        openstack network agent delete $id
    done

    openstack compute service list --os-compute-api-version 2.53 --host <host> -f value -c ID | while read id; do
        openstack compute service delete --os-compute-api-version 2.53 $id
    done

# Xóa node compute
Trước khi xóa compute node, cần migrate hoặc destroy các instance

Disable compute service trên node để tránh những instance mới được schedule vào:

    openstack compute service set <host> nova-compute --disable

Nếu có thể, hãy live migrate các instances đến một node khác

    openstack server list --all-projects --host <host> -f value -c ID | while read server; do
        openstack server migrate --live-migration $server
    done

Dừng toàn bộ dịch vụ trên node cần xóa:
