# 1. Deploy a registry

Pull image về một node:

      kolla-ansible pull -i multinode

Triên khai registry trên node đó:

      docker run -d \
      --network host \
      --name registry \
      --restart=always \
      -e REGISTRY_HTTP_ADDR=0.0.0.0:4000 \
      -v registry:/var/lib/registry \
      registry:2

Push image vào registry

      for img in $(docker images --format "{{.Repository}}:{{.Tag}}" | grep quay.io/openstack.kolla); do
      new=$(echo $img | sed 's#quay.io#10.2.2.209:4000#')
      docker tag $img $new
      docker push $new
      done

quay.io
   ↓
control1 pull toàn bộ image
   ↓
push vào local registry (control1)
   ↓
control2 / control3 / compute pull từ registry

Thêm vào file /etc/docker/daemon.json trên các node cho phép Docker pull/push image từ registry không dùng HTTPS (không có TLS certificate), thay ip bằng ip của server đã deploy registry:

       "insecure-registries": [
        "10.2.2.209:4000"
      ]
   
# 2. Ghi đè cấu hình
Kolla cho phép người dùng ghi đè cấu hình của các dịch vụ. Theo mặc định kolla sẽ
đọc cấu hình người dùng muốn ghi đè ở:
- /etc/kolla/config/<config file>
- /etc/kolla/config/<service name>/<config file>
- /etc/kolla/config/<tên server>/<service name>/<config file>

Ví dụ: ghi đè cấu hình nova cho compute1

- Tạo file nova.conf trong /etc/kolla/config/compute1/nova/

- Có thể cấu hình lại toàn bộ hoặc cấu hình các phần cần thay đổi so với file được tự động tạo ra trên node đó (/etc/kolla/nova-compute/nova.conf)

- Các cấu hình trong /etc/kolla/config/ sẽ được ghi đè lên các cấu hình được tự động tạo, sau đó dùng cho container chạy dịch vụ

# 3. Cấu hình file multinode
Các node có tên interface khác nhau sẽ cấu hình:

      control1 network_interface=ens3 neutron_external_interface=ens7
      control2 network_interface=ens3 neutron_external_interface=ens4
      control3 network_interface=ens3 neutron_external_interface=ens4

Bỏ cấu hình network_interface và neutron_external_interface trong file global.yml
